linux+CentOS+宝塔完成环境搭建和配置域名和SSL证书配置

2021-11-25

linux

对别人的意见要表示尊重。千万别说：“你错了。”——卡耐基

一直想写一下使用linux+CentOS+宝塔完成环境搭建和配置域名和SSL证书配置

今天正好有时间就记录一下

这里使用的服务器是：

阿里云轻量应用服务器

CentOS 7.6+Nginx+宝塔7.7.0+java1.8

1:首先安装宝塔

宝塔官网：https://www.bt.cn/bbs/thread-19376-1-1.html

使用Xshell工具输入：yum install -y wget && wget -O install.sh http://download.bt.cn/install/install_6.0.sh && sh install.sh

然后等他安装好之后会给咱们外网和内网的链接以及账号密码

然后去咱们的服务器开一下安全组：80/21/22/888

这里记录:

Xshell连接服务器无法用password登录，只能用public key的解决办法

我这里是阿里云的服务器 咱们进入阿里云服务器的后台找到远程链接的按钮切换到root目录下

修改/etc/ssh/sshd_config文件配置

找到PasswordAuthentication no 改成 PasswordAuthentication yes

然后重启sshd服务

CentOS 7 下重启服务不再通过 service 操作，而是通过 systemctl 操作

重启：systemctl restart sshd.service

然后就可以使用密码登录了

安装完之后我登录进去我这里选择的一键安装编译安装，也可以手动安装，这里没有使用docker管理

使用xshell查看宝塔账号信息：bt default

2:安装java环境

首先去官网下载jdk1.8:https://www.oracle.com/java/technologies/downloads/#java8

然后下载：jdk-8u311-linux-x64.tar.gz

我这里使用宝塔上传解压的 很是方便

然后打开咱们xshell 输入：vim /etc/profile 编辑咱们的环境变量

然后按住 i 编辑咱们的环境变量

然后输入：

JAVA_HOME=/home/java/jdk1.8.0_311

CLASSPATH=$JAVA_HOME/lib/

PATH=$PATH:$JAVA_HOME/bin

export PATH JAVA_HOME CLASSPATH

我这里安装的路径是/home/java/下的包名 根据自己装的目录更改

配置好之后按 Esc 退出编辑模式 光标消失之后 按 :wq 保存

配置好之后咱们重启配置文件 让配置生效 ：source /etc/profile

重启之后：java -version 查看java版本 如果出现一下版本号就证明咱们的java环境配置好了

3:启动jar文件

从这个文件上传咱们jar包 记得把jar对应端口打开 还有服务器的安全组 我这里是8651

咱们到对应的目录下面启动jar ：只输出错误信息到日志文件：nohup java -jar Demo.jar >liming Demo.log 2>&1 &

启动完成之后查看咱们的jar进程：ps -ef | grep java

这样就可以看到咱们启动的jar进程

启动出了一个问题：Exception in thread “main” java.lang.UnsupportedClassVersionError

这个问题是java环境不对 我项目是jdk11 刚才部署的是1.8 我换回来就好了 根据大家java版本部署

看下打印的日志是没有问题的 已经启动起来了

3:测试jar包接口是否调通

结果表示咱们的jar服务已经启动起来了并成功运行

目前咱们的公网ip和端口都暴露在外 咱们配置一下域名已经SSL证书

打开宝塔 网站->添加站点 把咱们买好的域名配置上

4：测试域名是否能够调通

如图使用咱们的域名访问也是没有问题的 这里记录一下因为我改过Nginx和站点的默认端口所以我这个端口是可以访问到 因为Nginx和站点默认都是80端口 如果是咱们自定义端口 得去配置文件里面改一下 然后重启咱们的Nginx。

修改这两个文件夹的配置 把 listen 80; 改成咱们自定义端口 我这里改成86。咱们的域名记得备案！

然后咱们的域名访问接口就好了，但是http是不安全链接 咱们的端口还暴露在外 咱们还需要配置SSL证书
